Minecraft =/= Reality

In Minecraft the formulas can change a bit from the reality. Here’s the documentation to make those calculations in Minecraft with no errors.

1. Volume

In real life, the volume is calculated by the following formula Volume=Width∗LengthVolume = Width * LengthVolume=Width∗Length﻿

However in our case, this is a 3d plan, so the formula is Volume=Width∗Length∗HeightVolume = Width * Length * HeightVolume=Width∗Length∗Height﻿

To calculate the volume from coordinates, x1, y1, z1 and x2, y2, z2 we’ll use vectors.

Vectorx=x2−x1Vector_x = x2 - x1Vectorx​=x2−x1﻿

Vectory=y2−y1Vector_y = y2-y1Vectory​=y2−y1﻿

Vectorz=z2−z1Vector_z = z2 -z1Vectorz​=z2−z1﻿

In real life, it would work, but not in Minecraft. In Minecraft, the coordinate “0” is also counted as 1 block. For example to calculate the number of blocks from x1 = 0 and x2 = 3, there are 4 blocks (x2−x1)+1(x2 - x1) + 1(x2−x1)+1﻿

if we apply this to our previous vectors, it would work. However it must be a positive number.

Vectorx=∣(x2−x1)+1∣Vector_x = |(x2 - x1) + 1|Vectorx​=∣(x2−x1)+1∣﻿

Vectory=∣(y2−y1)+1∣Vector_y = | (y2-y1) + 1 |Vectory​=∣(y2−y1)+1∣﻿

Vectorz=∣(z2−z1)+1∣Vector_z = |(z2-z1) + 1|Vectorz​=∣(z2−z1)+1∣﻿

and now, we can apply the volume formula Volume=Vectorx∗VectorZ∗VectoryVolume = Vector_x * Vector_Z * Vector_y Volume=Vectorx​∗VectorZ​∗Vectory​﻿

This program in python : 

vector_x = x2 - x1
if vector_x < 0:
	vector_x *= -1
vector_x += 1

vector_y = y2 - y1
if vector_y < 0:
	vector_y *= -1
vector_y += 1

vector_z = z2 - z1
if vector_z < 0:
	vector_z *= -1
vector_z += 1

volume = vector_x * vector_z * vector_y

2. Perimeter

In real life, the perimeter is calculated by the following formula Perimeter=(Length+Width)∗2Perimeter = (Length + Width) * 2Perimeter=(Length+Width)∗2﻿

However in our case, this is a 3d plan, so the formula is Perimeter=((Length+Width)∗2)∗HeightPerimeter = ((Length + Width) * 2) * HeightPerimeter=((Length+Width)∗2)∗Height﻿

To calculate the perimeter from coordinates, x1, y1, z1 and x2, y2, z2 we’ll use vectors.

Vectorx=x2−x1Vector_x = x2 - x1Vectorx​=x2−x1﻿

Vectory=y2−y1Vector_y = y2-y1Vectory​=y2−y1﻿

Vectorz=z2−z1Vector_z = z2 -z1Vectorz​=z2−z1﻿

This is where it is different from the volume. For the perimeter we don’t add 1 to vectors except for VectoryVector_yVectory​﻿

The vectors also need to be positive.

Vectorx=∣x2−x1∣Vector_x = |x2 - x1|Vectorx​=∣x2−x1∣﻿

Vectory=∣(y2−y1)+1∣Vector_y = | (y2-y1) + 1 |Vectory​=∣(y2−y1)+1∣﻿

Vectorz=∣z2−z1∣Vector_z = |z2-z1|Vectorz​=∣z2−z1∣﻿

and now, we can apply the perimeter formula Perimeter=((Vectorz+Vectorx)∗2)∗VectoryPerimeter = ((Vector_z + Vector_x)*2)*Vector_yPerimeter=((Vectorz​+Vectorx​)∗2)∗Vectory​﻿

This program in python : 

vector_x = x2 - x1
if vector_x < 0:
	vector_x *= -1

vector_y = y2 - y1
if vector_y < 0:
	vector_y *= -1
vector_y += 1

vector_z = z2 - z1

	vector_z *= -1

4. Blocks <=> Chunks

Easy one.

16 Blocks = 1 chunk

1 chunk = 16 blocks

So

InChunks=blocks/16InChunks = blocks / 16InChunks=blocks/16﻿

InBlocks=chunks∗16InBlocks = chunks * 16InBlocks=chunks∗16﻿

In python

InChunks = blocks * 16
InBlocks = chunks / 16

If it’s north or west the value CCC﻿ equals to -310

If it’s south or east it is equal to 310.
